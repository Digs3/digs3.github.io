Vue.JS

The Progressive JavaScript Framework

What is Vue.JS?

Vue.JS is a JavaScript framework for building reactive user interfaces.

How to use Vue.JS

To use Vue.JS, simply add the vue.js file to your page with a script tag.

Vue Instance Options

When you initialize Vue as shown above, you can set a variety of different options. A description of each option is provided below:

Option Description
el Existing DOM element to mount on
data Data object for Vue instance
methods Methods to be mixed into Vue instance
computed Computed properties to be mixed into Vue instance
watch Watch properties for changes.

Directives

Computed Properties

Computed properties are automatically updated when any variables they rely upon are changed. You can insert computed properties into your HTML just like you would any data from the Vue instance.

Computed properties are different than methods because they are cached. A computed property's method is only ran when the dependencies of the computed property changes. Methods are always called. As such, computed properties are generally preferred to methods, especially when you're doing api calls. 

                var vm = new Vue({
                  el: '#example',
                  data: {
                    message: 'Hello'
                  },
                  computed: {
                    reversedMessage: function () {
                      return this.message.split('').reverse().join('')
                    }
                  }
                })

Computed properties can have getters and setters. By default you only have a getter method. You can create a setter method like this: 

                computed: {
                  fullName: {
                    // getter
                    get: function () {
                      return this.firstName + ' ' + this.lastName
                    },
                    // setter
                    set: function (newValue) {
                      var names = newValue.split(' ')
                      this.firstName = names[0]
                      this.lastName = names[names.length - 1]
                    }
                  }
                }

Events

Use the v-on directive to listen to DOM events and react.

v-on can also accept a method.

You can refer to the native DOM event with the keyword event. 

                methods: {
                  greet: function (event) {
                    alert('Hello ' + this.name + '!')
                    if (event) {
                      alert(event.target.tagName)
                    }
                  }

When using inline Javascript you can use the special $event variable to access the original DOM event. 

                
              
                methods: {
                  warn: function (message, event) {
                    if (event) event.preventDefault()
                    alert(message)
                  }
                }

Filters

Filters are helpers. Vue provides the capability of using filters, but you create the filters yourself. Use computed properties instead of filters unless it makes more sense to use a filter.

How to build a filter

Creating a global filter

                Vue.filter('uppercase', function(value) {
                  return value.toUpperCase();
                });

Creating a local filter

                new Vue({
                  filters: {
                    lowercase: function() {
                      return value.toLowercase();
                    }
                  }
                })

How to use a filter

                
{{ title | uppercase }}

Filters can be chained

                
{{ title | uppercase | lowercase }}
