HTML5 (HyperText Markup Language)

The standard markup language used to create web pages

HTML Element Reference

What is HTML5?

HTML (Hypertext Markup Language) is the markup language for describing web pages. Writing HTML is as simple as adding tags to a text document. These tags create the HTML elements in what will become a web page.

In the following example, the <h1> and <p> tags are used to create two HTML elements.

        	  		<h1>This is a level 1 heading element</h1>
        	  		<p>This is a paragraph element</p>

The <p> element is used to create paragraph text and the <h1> element creates a top level heading. There are also <h2> through <h6> heading elements. These two types of elements are used throughout this very article to form its structure.

HTML5 is the latest version of HTML and with it came many new semantic elements, support for dynamic pages, new form input types, and other new elements like canvas, video, and audio.

Basic Structure of an HTML page

The following HTML markup outlines the basic structure of an HTML page:

        	  		<!DOCTYPE html>
        	  		<html>
        	  		  <head>
        	  		    <title>Title of Page</title>
        	  		  </head>
        	  		  <body>
        	  		  	<p>content goes here</p>
        	  		  </body>
        	  		<html>

All HTML5 pages begin with the HTML5 <!DOCTYPE> which you will learn about next. All HTML pages get wrapped in an <html> element. Within the <html> element you will find the <head> and <body> elements. The <head> element is where you include metadata about your page and the <body> is where you place your content.

If you copy and paste the previous markup into a text editor such as Notepad and save it with an extension of .html, you will have created a simple HTML5 page that can be loaded into a Web browser.

HTML5 <!DOCTYPE>

The <!DOCTYPE> declaration represents the document type. The browser uses this to determine which version of HTML/XHTML the document is using to display the page correctly. This declaration must appear only once, at the top of the HTML page.

For HTML5, the doctype is <!DOCTYPE html>

Semantic Elements

Semantic elements are elements that clearly describe their meaning to both the browser and the developer. For example, the <p> element tells you and the browser that it contains paragraph text. The <h1> element tells you and the browser that it contains a heading. These are semantic elements and most elements in HTML5 are semantic.

Examples of non-semantic elements are div and span. These elements tell us nothing about their content. Non-semantic elements should be avoided when possible, but often come in useful for grouping content solely for styling purposes (see CSS).

Layout Elements

These six elements should be used to create the structure of your page's layout.

Tag Description
<section> Defines a section in a document.
<article> Specifies independent, self-contained content.
<header> Specifies a header for a document or section.
<footer> Specifies a footer for a document or section.
<nav> Defines a set of navigation links.
<aside> Defines some content aside from the content it is placed in.

Custom Data Attributres

data- attributes allows us to store extra information on standard, semantic HTML elements.

Example:

        				<ul>
        				  <li data-animal-type="bird">Owl</li>
        				  <li data-animal-type="fish">Salmon</li> 
        				  <li data-animal-type="spider">Tarantula</li> 
        				</ul>

The values of these attributes can be accessed in JavaScript using the getAttribute() method or through the dataset property.

Audio & Video

<audio> : a standard way to embed an audio file on a web page

        				<audio controls>
        				  <source src="song.ogg" type="audio/ogg" />
        				  <source src="song.mp3" type="audio/mpeg" />
        				  Not Supported
        				</audio>

Multiple sources -> the browser will use the first recognized format.
