Integrating Foundation into Laravel

What is Laravel

Laravel is a free, open-source MVC PHP web framework.

Features

  • Modular packaging system with a dedicated dependency manager
  • Different ways for accessing relational databases through Routing
  • Utilities that aid in application deployment and maintenance
  • Orientation toward syntactic sugar
  • Easy authentication by providing a simple & easy to use interface

Laravel uses composer to manage dependencies.

Setting up Laravel

Laravel requires composer (a PHP dependency manager), so install that first.

Install laravel with composer global require "laravel/installer".

How to use Laravel

Create a laravel project with laravel new <projectName>

The Web server's document root should point to the /public directory

File Structure

The project/app folder contains the Models and Controllers and most of the logic.

The project/app/User.php file is an example of a Model.

The project/app/Http folder contains the Controllers directory for the controller code. The Middleware directory contains code that will be processed before the Http request and routes file. Code that is replicated very often is placed here.

The project/bootstrap folder is where Laravel gets bootstrapped and started up.

The project/config directory contains the files that contain the configuration settings for Laravel and different packages.

The project/database directory contains migrations and seeds.

The project/public directory contains assets and is publicly accessible. All requests go through index.php in this directory.

The project/resources directory contains assets like Sass code and our Views. Views are our html pages that the user sees.

The project/routes directory contains all the route files.

The project/storage folder contains logs and other storage

The project/tests folder is used for PHP unit testing.

The project/vendor folder contains 3rd party vendor libraries.

Models

A model relates to a table. In most cases you'll have one model per table.

  • Models are stored in the /project/app directory as .php files.
  • Model names should be in singular form and capitalized (e.g., Post.php or User.php).
  • Models generally have a timestamp created with columns named 'created_at' and 'updated_at'

Creating Models

Generate a Model with Artisan

php artisan make:model Customer

Use --migration or -m after the command to generate a database migration.

Laravel's Eloquent ORM assumes these things:

  • The table name is the plural form of the model name
  • The table has a primary key named id
  • The primary key is an incrementing integer value
  • Foreign key names are the referenced table's name + _id
  • Tables have created_at and updated_at columns
Customize Eloquent's Behavior
  • Use protected $table = 'table_name'; to override the table name
  • Use the $primarykey property to override the primary key name
  • Set the $incrementing property to false to use a non-incrementing or non-numeric primary key
  • Set the $timestamps property to false to stop Eloquent from automatically managing these columns.
  • Set the CREATED_AT or UPDATED_AT constants to customize the names of the columns used to store timestamps

Connecting to Database

First, create your database in MySQL.

Open the .env file in the /project directory and edit the database environment variables.

  • DB_HOST - db server IP
  • DB_PORT - db server Port
  • DB_DATABASE - db name
  • DB_USERNAME - db username
  • DB_PASSWORD - db password

Open the database.php file in the /project/config directory. This file lets you change the settings for the database.

Relationships

Eloquent supports relationships as defined methods on your model classes.

Eloquent assumes names of keys and more.

Examples:

        		// One To One
        		class User extends Model {
        			public function phone() {
        			  return $this->hasOne('App\Phone'); }
        		}
        
        		// One To One (Inverse)
        		class Phone extends Model {}
        			public function user() {
        			  return $this->belongsTo('App\User'); } 
        		}
        
        		// One To Many
        		class Post extends Model {
        			public function comments() {
        			  return $this->hasMany('App\Comment'); }
        		}
        
        		// One To Many (Inverse)
        		class Comment extends Model {
        			public function post() {
        			  return $this->belongsTo('App\Post'); }
        		}
        
        		// Many To Many
        		class User extends Model {
        			public function roles() {
        			  return $this->belongsToMany('App\Role'); }
        		}
        
        		// Many To Many (Inverse)
        		class Role extends Model {
        			public function users() {
        			  return $this->belongsToMany('App\User'); }
        		}

The belongsToMany() function automatically assumes the intermediary table name (table names in alphabetical order separated by a _) unless you explicitly specify it as the 2nd parameter:

$this->belongsTo('App\Post', 'post_tag');

The belongsToMany() can also take the column names for the current and joining models as the 3rd and 4th parameters:

$this->belongsTo('App\Post', 'post_tag', 'post_id', 'tag_id');

Once the relationship is created, you can access the relating tables as follows:

$post->category->name

Migrations

Migrations are a way of storing your database schema in files in addition to the database. This allows other people working on the project to have the database schema.

Migrations are a way for laravel to create or edit our database tables for us

Migrations are stored in the /project/database/migrations directory.

Run php artisan migrate to create the database tables from the migrations.

Creating Migrations

You can create a migration when you make a model with the -m or --migration modifier.

php artisan make:model Post -m

You can also create a migration using the make:migration artisan command

php artisan make:migration create_users_table --create=users

The --create modifier will automatically create the create and drop functions with the chosen name for the table.

To create a migration that adds a column to an existing table use the --table modifier

php artisan make:migration add_category_id_to_posts --table=posts

Modifying Migrations

Migrations are stored in the /database/migrations folder of your project.

Creating a table

        		Schema::create('posts', function (Blueprint $table) {
                    $table->increments('id');
                    $table->string('title');
                    $table->text('post');
                    $table->timestamps();
                });

Create a foreign key in the table:

        			$table->integer('tag_id')->unsigned();
                    $table->foreign('tag_id')->references('id')->on('tags');

Adding a column to a table

        		class AddSlugToPosts extends Migration
        		{
        		    public function up()
        		    {
        		        Schema::table('posts', function(Blueprint $table) {
        		            $table->string('slug')->unique()->after('body'); });
        		    }
        		    public function down()
        		    {
        		        Schema::table('posts', function(Blueprint $table) {
        		            $table->dropColumn('slug') })
        		    }
        		}

Column Modifiers

->nullable() Allow NULL values to make a field optional
->default(value) Set a "default" value
->first() Place the column "first" in the table
->after('column') Place the column "after" another
->unsigned Set integer columns to UNSIGNED
->unique Create an Index on the column

Views

Views contain the HTML served by your application. Views are stored in the resources/views directory.

Views are html documents saved as .blade.php files as they make use of the Blade templating engine.

You can create subdirectories to organize your views.

Blade

Blade is the templating engine provided with Laravel.

Blade lets you output variables using a simple syntax:

{{ $customer->name }}

{{ $fullname }}

Control Structures

Use if statements with Blade:

        		  @if($var1 == 'Hamburger')
        		  	I love hamburgers<br>
        		  @endif

Use foreach statements with Blade:

        		  @foreach($orders as $order)
        		  	<li>{{ $order->name }}</li>
        		  @endforeach

Layouts

Create a Layout by adding @section and @yield directives to your HTML markup.

@section defines a section of content

        		@section('content')
        		  <div>HTML content</div>
        		@endsection

Sections should now end with either @stop or @show. @stop will only show the section and @show will show the section plus any content for the section residing in the template.

        		@section('title', ' | Homepage')

@yield displays the contents of a given section

        			@yield('content')

Place your layout in the resources/views/layouts/ directory

Create a child view using the @extends directive to specify which layout the child view should inherit

For example, you may create a main.blade.php file that contains the HTML markup that is the similar across all pages. This will probably include your page title, the nav bar, and any header or footer elements. Place a @yield('content') directive where the content should go.

All content pages will include an @extends('main') directive at the top followed by a @section('content') directive. This will load the main.blade.php file with the content of the content page.

Partials

Partials are a way to extract code out into its own file in order for it to be repeated or for organization.

Partials should begin with an underscore. Example: _head.blade.php. This is not required but it can be helpful.

Include a partial with the @include directive

        		  @include('partials._head')

Forms

Form Helpers

Form Helpers are not included in Laravel 5. To add them, go to laravelcollective.com and browse to HTML & Forms.

CSRF Protection

All forms should include a hidden input field named _token that has the value of csrf_token(). Use the csrf_field() function to automatically generate the entire input field.

Accessing PUT and DELETE

To access PUT and DELETE when using CRUD, set the form method to POST and add the following input field with the value set to PUT or DELETE:

<input type="hidden" name="_method" value="PUT">

Controllers

Controllers are stored in the project/app/Http/Controllers directory.

The Controller.php file is the controller base to be extended upon should you make your own controller.

Generate a controller with artisan

php artisan make:controller CustomerController

Use the --resource modifier to create a resource controller.

How to build a controller

This sample controller will handle the routing of static pages

Create your PagesController.php file in the Controllers directory.

        			<?php
        			  // All controllers need to be namespaced
        			  namespace App\Http\Controllers;
        
        			  class PagesController extends Controller {
        			    public function getIndex() {
        				  # process variable data
        				  # talk to the model
        				  # receive from the model
        				  # compile or process data
        				  # pass data to the correct view
        				  return view('pages.welcome');
        			    }
        
        			    public function getAbout() {
        			      $first = 'John';
        			      $last = 'Doe';
        			      $full = $first . " " . $last;
        			      $email = 'john.doe@gmail.com';
        			      $data = [];
        			      $data['email'] = $email;
        			      $data['fullname'] = $full;
        			      return view('pages.about')->with("data", $data);
        			      // Shortcut: return view('pages.about')->withData($data)
        			      // You can add multple withs:  ->withEmail($email)->withName($name)->withAge($age);
        			    }
        			  }
        			?>

Returning a View from a Controller

Use the global view helper to return views from controllers

The second argument is an associative array of data that should be made available to the view.

        	  	  $customer = Customer::find($id);
        	  	  return view('customer', ['customer' => $customer]);
        
        	  	  // Using the compact function with name of variable
        	  	  return view('greeting', compact('customer'));

Alternatively, you can use the with method to add individual pieces of data to the view:

        	  	  return view('greeting')->with('name', 'Victoria');
        
        	  	  return view('pages.about')->with('fullname', $fullname);
        	  	  // Shorthand and passing multiple values
        	  	  return view('pages.about')->withFullname($fullname)->withEmail($email);
        
        		  // Pass an associative array
        	  	  return view('pages.about')->withData($data);

The first argument is the name you want to access the variable as in the view. The second argument is what it should be equal to.

Resource Controllers (CRUD)

Create a resource controller with php artisan make:controller PostController --resource

The --resource tag creates a resource controller which creates basic CRUD commands.

Resource Controller Functions

index() Lists all items (posts, customers, etc)
create() Shows a create form
store() Stores a new item
show() Displays a single item
edit() Show an edit form
update() Update an existing item
destroy() Deletes an item

create()

Just return the view for creating (e.g., submit post form)

store()

Comes in as a post request

Validate the data, store in the db, redirect to other page

1. Validate the Data

Laravel includes validation methods with all controllers

Use the $this->validate() function to validate data.

When validation fails, the create() function is automatically invoked with flash data called errors that will contain the error messages.

The $this->validate() function takes 2 arguments: a request, and an array of validation rules:

Check the validation documentation for a list of available rules.

$this->validate($request, array('title' => 'required|max:255', 'body' => 'required')).

For client-side javascript validation, check out the Parsley library.

Save parsley.min.js inside your /project/public/js folder and the parsley.css in your /project/public/css folder if you decide to use it.

        			@if (count($errors) > 0)
        		  	  @foreach ($errors->all() as $error)
        		  	    {{ $error }}
        		  	  @endforeach
        		  	@endif
2. Store in DB

To store in the db, use Eloquent.

Add the post model with a use clause at the top of the controller. use App\Post;

        		    $post = new Post;
        		    $post->title = $request->title;
        		    $post->body = $request->body;
        		    $post->save();
3. Redirect

To redirect do return redirect()->route('posts.show', $post->id);

show($id)

        			public function show($id)
        			{
        		  		$post = Post::find($id);
        		  		return view('posts.show')->withPost($post);
        		  	}

edit($id)

        			public function edit($id)
        			{
        		    	$post = Post::find($id);
        		    	return view('posts.edit')->withPost($post);
        		    }

update(Request $request, $id)

Like store(), we want to validate the data, save the data to the database, set flash data with success message, and redirect with flash data to posts.show

        			public function update(Request $request, $id)
        		    {
        		        // validate the data
        		        $this->validate($request, [
        		                'title' =>  'required|max:255',
        		                'body'  =>  'required'
        		            ]);
        
        		        // store in the database
        		        $post = Post::find($id);
        		        $post->title = $request->input('title');
        		        $post->body = $request->input('body');
        		        $post->save();
        
        		        // Set a session flash message to display on the next request
        		        Session::flash('success', 'The blog post was edited successfully.');
        
        		        // redirect to another page
        		        return redirect()->route('posts.show', $post->id);
        		    }

Middleware

Middleware provides a convenient mechanism for filtering HTTP requests entering the application. It allows you to run code before the application receives requests.

Think of middleware as layers that surround your app. Requests and responses have to pass through the layers on their way in and out.

Examples of middleware included with Laravel: Session, Authentication

You can create your own middleware.

Sessions

Sessions provide a way to store information about the user across multiple requests. This storage is temporary (2 hours in Laravel) unless you save it to the database.

Configure sessions through the config/session.php file.

Add Use Session; to your using clauses to use the Session facade.

Check if a variable exists with Session::has('variable')

Session::put('key', 'value') sets a variable in the session.

Use {{ Session:get('key') }} to get the value of a session variable.

Flash Data

Session::flash('key', 'value') creates a variable in the session for a single request.

Session::flash('success', 'The blog post was successfully saved.')

Data stored in the session using the flash method will only be available during the subsequent HTTP request.

        			$request->session()->flash('key', 'value');

Routings

Routes are located in the project/routes directory. The routes/web.php file defines routes that are for your web interface.

  • Place more specific routes at the top as laravel will use the first applicable route.
  • Don't place functions in your routes. Instead, route to a controller.
        			<?php
        			  // Basic Route
        			  Route::get('welcome', function() {
        			  	echo 'Hello Laravel';
        			  }
        			  // Passing in parameters
        			  Route::get('hello/{name}', function($name) {
        			  	echo 'Hello ' . $name;
        			  }
        			  // Using Controllers
        			  Route::get('about', 'PagesController@getAbout');
        			  Route::get('/', 'PagesController@getIndex');
        			  
        			  // Resource Controller Routes
        			  Route::resource('posts', 'PostController'); 
        			  Route::resource('photo', 'PhotoController', ['only' => ['index', 'show']]);
        			  Route::resource('photo', 'PhotoController', ['except' => ['create', 'store', 'update', 'destroy']]);
        
        			  // Restful (CRUD) Routes
        			  // Create Item
        			  Route::post();
        			  // Read Item
        			  Route::get();
        			  // Update Item
        			  Route::put();
        			  // Delete Item
        			  Route::delete();
        
        			?>

To list all routes use the command php artisan route:list

To support PUT, PATCH, or DELETE routes add a hidden _method field to the form with the value set as the HTTP requst method (e.g., PUT). This is performed automatically if you use the Form Helpers from the Laravel Collective.

Authentication

Basic Authentication Information

Laravel comes with several pre-built authentication controllers located in the App\Http\Controllers\Auth namespace.

RegisterController handles new user registration
LoginController handles authentication
ForgotPasswordController handles e-mailing links for resetting passwords
ResetPasswordController contains the logic to reset passwords

How to create authentication

The following command will create all the authentication scaffolding files in the /project folder. This includes a layout view, registration and login views, as well as routes for all authentication end-points. A HomeController will also be generated to handle post-login requests at your application's dashboard.

php artisan make:auth

You need to have a database that has the same name as your application.

If the database is created, the following command will create a users and passwords_resets table in your database:

php artisan migrate

Laravel will have automatically setup the entire authentication system. Run php artisan serve when loading the default Welcome view and see it in action

Retrieving the Authenticated User

        		  // Get the currently authenticated user
        		  $user = Auth::user();
        
        		  // Get the currently authenticated user's ID
        		  $id = Auth::id();
        
        		  if (Auth::check()) {
        		    // The user is logged in...
        		  }
        
        		  if (Auth::guest()) {
        		    // The user is not logged in...
        		  }

Manually Authenticating Users

You can manually authenticate users by removing the included authentication controllers.

You can access Laravel's authentication services via the Auth facade.

The following code shows how to use the Auth::attempt() method to manually authenticate a user:

        		use Illuminate\Support\Facades\Auth;
        
        		class LoginController extends Controller
        		{
        		    public function authenticate()
        		    {
        		        if (Auth::attempt(['email' => $email, 'password' => $password])) {
        		            // Authentication passed...
        		            return redirect()->intended('dashboard');
        		        }
        		    }
        		}

Logging Out

        		Auth::logout();

Remembering Users

How: Pass a boolean value as the second argument to the attempt() method.

Note: Your users table must include the string remember_token column.

        		$remember = true;
        		if (Auth::attempt(['email' => $email, 'password' => $password], $remember)) {
            		// The user is being remembered...
        		}

Guards

Guards allow you to manage the authentication for separate parts of your application using entirely separate authentication models or user tables.

        		if (Auth::guard('admin')->attempt($credentials)) {
         			//
        		}

Protecting Routes

To only allow authenticated users access to a route, use the auth middleware.

        		Route::get('profile', function () {
        		    // Only authenticated users may enter...
        		})->middleware('auth');

If you're using controllers, you can call the middleware method from the controller's constructor instead.

        		public function __construct()
        		{
        		    $this->middleware('auth');
        		}

Slugs

A slug is a simplified version of a string, typically URL-friendly.

Use slugs to change your urls from http://example.com/post/1 to http://example.com/post/my-dinner-with-alice

To use slugs, first add a column to your database table named slug. Index this column with the ->unique() method to speed up your database. You can do all of this using migrations.

Create a route for your slug. The following example includes a regular expression to constrain the route format.

        	  		Route::get('blog/{slug}', ['as' => 'blog.single', 'uses' => 'BlogController@getSingle'])->where('slug', '[\w\d\-\_]+');

Create the controller function for your slug route.

        	  		public function getSingle($slug) {
        		    	$post = Post::where('slug', '=', $slug)->first();
        		    	return view('blog.single')->withPost($post);
        		    }

And finally, create the View for the slug.

Image Upload

Use the Image Intervention library for image handling and manipulation.

Store images in your project/public folder or in the project/storage folder. If you use the project/storage folder you will need to use the Laravel API to access the images.

Create the Image Upload Form

        			{{ Form::label('featured_image', 'Upload Featured Image:')}}
                    {{ Form::file('featured_image') }}

Set the Form to accept files

When using Form Helpers, simply set the files value to true:

        			 {!! Form::open(['route' => 'posts.store', 'files' => true]) !!}

When not using Form Helpers, add the following to your form tag

enctype="multipart/form-data"

Create a Migration to add an Image column to your table

        			$table->string('image')->nullable()->after('slug');

Save the uploaded image

Use the time() function to create a timestamp to give the image a unique name.

        	        if($request->hasFile('featured_image')) {
        	            $image = $request->file('featured_image');
        	            $filename = time() . '.' . $image->getClientOriginalExtension();
        	            $location = public_path('images/' . $filename);
        	            Image::make($image)->resize(800,400)->save($location);
        	            $post->image = $filename;
        	        }

Display the image

        			<img src="{{ asset('images/' . $post->image) }}" height="400" width="800">
