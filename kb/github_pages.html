GitHub Pages (Websites for you and your projects)

Official Website / A Guide To Using Github Pages

GitHub Pages is a free static site hosting service. Use it to host your personal, organization, or project pages directly from a GitHub repository.

Using GitHub Pages will give you practice using Git and GitHub.

Types of Pages

There are two basic types of GitHub Pages

  • User Pages - for hosting a personal website or an index of your projects.
  • Project Pages - for hosting a project, of which you have many. You can create multiple project pages per account.

User Pages live in a special GitHub repository with the name username.github.io. The master branch of the repository is used to build and publish the site.

Project Pages are kept in the same repository as the project they are for, except that the website content is stored in a specially named gh-pages branch. The output for this branch will be available under username.github.io/project

Getting Started

Create a GitHub account if do not yet have one.

  1. In GitHub, create a new repository.
    • If this is your User site, name this repository username.github.io, where username is your username on GitHub.
    • If this is a Project site, name this repository whatever you like (e.g., ProjectName).
    • Make the repository Public
    • Check "Initialize this repository with a README"
  2. At the command line, create a directory for the site (mkdir) and navigate to it (cd)
            $ mkdir /path/to/source-code
        $ cd /path/to/source-code
  3. Download (clone) your newly created GitHub repository and go inside it 
            $ git clone https://github.com/username/repositoryName.git
        $ cd repositoryName
  4. If this is a Project site, create a new branch called gh-pages and check it out 
            $ git checkout -b gh-pages
  5. Copy your already created site files into this directory or start a new site from scratch.
  6. Add, commit, and push your new branch to GitHub. branchName will be master for a User site and gh-pages for a Project site. 
            $ git add .
        $ git commit -a -m "Adding my existing site."
        $ git push origin branchName

gulp-gh-pages

gulp-gh-pages is a plugin for gulp that lets you easily deploy your sites to Github pages.

Install gulp-gh-pages with npm install --save-dev gulp-gh-pages.

Define a deploy task in your gulpfile.js to push your contents to Github pages

        var ghPages = require('gulp-gh-pages');
        
        var options = { 
            remoteUrl: "https://github.com/Digs3/digs3.github.io.git",
            branch: "gh-pages"};
        gulp.task('deploy', function () {
          return gulp.src("./dist/**/*")
            .pipe(ghPages(options))
        });

Run your deploy task with gulp deploy in console.
