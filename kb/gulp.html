Gulp (The Streaming Build System)

Official Website / Gulp for Beginners

Gulp is a build system that can automate and enhance your workflow by compiling sass files, uglifying and compressing js files, providing a local server, and much more.

Gulp uses streams.

Getting Started

Gulp runs on Node.js so install it first.

  1. Install Gulp with npm install --global gulp-cli
  2. Initialize your project directory with npm init (creates package.json)
  3. Install gulp in your project devDependencies with npm install --save-dev gulp (creates node_modules folder)
  4. Create a gulpfile.js at the root of your project
  5. Run gulp.

Defining Tasks & Configuring Plugins

With Gulp set up, we can now configure it to perform specific tasks by editing gulpfile.js.

Defining a Task
  1. First we must require Gulp in the gulpfile. 
            var gulp = require('gulp');
  2. Now we can define a task. 
            gulp.task('hello', function() {
          console.log('Hello World');
        });
  3. We can now run this task with gulp hello in the command line.
Using a plugin
  1. To use a gulp plugin it must first be installed. 
            npm install gulp-sass --save-dev
  2. Then it must be required in the gulpfile. 
            var sass = require('gulp-sass');
  3. Finally, configure a task to use it. 
            gulp.task('sass', function(){
          // Get the files to use for the task
          return gulp.src('app/scss/**/*.scss') 
            // Convert Sass to CSS using gulp-sass on the files
            .pipe(sass()) 
            // Place converted files in destination
            .pipe(gulp.dest('app/css'))
        });

Running tasks automatically

Gulp provides the watch() method to automatically run tasks on file changes. For example, gulp can run the sass task when it detects changes in any .scss file.

To use watch(), simply supply it with a glob (a matching file pattern) and a list of tasks to run when any files in your file pattern changes.

        gulp.task('watch', function(){
          gulp.watch('app/scss/**/*.scss', ['sass']); 
        })
What are globs?

Globs are matching file patterns for files. The most common glob patterns are:

  • *.scss will match any file ending in .scss in the root folder.
  • **/*.scss will match any file ending in .scss in the root folder and any child directories.
  • !not-me.scss will exclude not-me.scss from the match
  • *.+(scss|sass) will match any file ending with .scss or .sass in the root folder.

Dependent sub-tasks

This code will start both sass and browserSync concurrently and when complete will run watch

        gulp.task('watch', ['browserSync', 'sass'], function (){
          gulp.watch('app/scss/**/*.scss', ['sass']); 
        });

The default task

You can name a task default and it will run when you just run gulp without a task name at the command prompt.

Gulp Plugins

  • gulp-sass : Sass Compiler
  • browser-sync : Live CSS Reload & Browser Syncing
  • gulp-autoprefixer : Parse CSS and add vendor prefixes to rules
  • gulp-uncss : Remove unused CSS selectors
  • gulp-babel : JS Transpiler (Compiles ES2015 JS to ES5)
  • gulp-concat : Concatenates files
  • gulp-uglify : JS Minifier
  • gulp-cssnano : CSS Minifier
  • gulp-imagemin : Image Minifier
  • gulp-if : Conditionally run a task

Cleaning up generated files

Use del or rimraf for cleaning. del adds support for multiple files, globbing, and protects you from deleting your working directory.
