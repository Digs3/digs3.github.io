Workflows

Node/Grunt/Bower Workflow

  1. Create the project directory & structure (src, dist, test folders)
  2. Run npm init to create & configure package.json
  3. Run npm install grunt --save-dev to install grunt ∓ add it as a devDependency
  4. Create & configure gruntfile.js
  5. Run bower init to create & configure bower.json
  6. Install bower packages with bower install --save <pkg>

Yeoman Workflow

  1. Install the yeoman generator using npm install -g <generator>
  2. Create a directory for the site and run the generator with yo <generatorName>
  3. Install dependencies with npm install and bower install (is this necessary?)
  4. Test server with grunt serve
  5. Install additional bower components (e.g., foundation-sites)
  6. Add "$(document).foundation();" to app.js to use Foundation's JS abilities

Foundation CLI Workflow

Install Foundation CLI with npm install -g foundation-cli

  • Basic Template
  • ZURB (Advanced) Template
  1. Run foundation new to create a new project (this will create the project dir)
  2. Manually set up a template with git clone and then install dependencies
    • git clone <template-url> projectname
    • npm install
    • bower install
  3. Run npm start to run Gulp (just running gulp seems to work too)
  4. Run npm run build to create compressed, production-ready assets

Better Method to avoid the large, nested node_modules folder

  • Copy the advanced master template to the project directory
  • run npm install and bower install
  • Run npm start to run it
