WPF XAML

What is WPF?

WPF stands for Windows Presentation Foundation. It is Microsoft's next generation GUI framework for Windows-based applications created with the .NET framework 3.0 and higher.

WPF combines application UIs, 2D graphics, 3D graphics, documents and multimedia into one single framework. Its vector based rendering engine uses hardware acceleration of modern graphic cards. This makes the UI faster, scalable and resolution independent.

WPF separates the appearance of an user interface from its behavior. The appearance is generally specified in the Extensible Application Markup Language (XAML), the behavior is implemented in a managed programming language like C# or Visual Basic. The two parts are tied together by databinding, events and commands.

What is XAML?

XAML stands for Extensible Application Markup Language. It is the markup language used for defining the interface in WPF.

Use property element syntax to put a complex object as content.

        			<Button>
        			  <Button.Content>
        			    <Image Source="Images/OK.png" Width="50" Height="50" />
        			  </Button.Content>
        			</Button>

Markup Extensions

Markup extensions are dynamic placeholders for attribute values in XAML. They resolve the value of a property at runtime. Markup extensions are surrouded by curly braces (Example: Background="{StaticResource NormalBackgroundBrush}").

        				<TextBox x:Name="textBox"/>
        				<Label Content="{Binding Text, ElementName=textBox}"/>

Accessing Controls in Code

To access a control it needs a name defined.

<Button x:Name="ApplyButton" />

To give a control an event, simply type the event into the control's definition and press Tab to create the event handler

<Button x:Name="ApplyButton" Click="ApplyButton_Click" />

Dependency Properties

Dependency properties extend upon CLR properties and add specific functionalities. The purpose of dependency properties is to provide a way to compute the value of a property based on the value of other inputs.
