GitHub (Web-based Git repository hosting service)

Official Website

A platform for hosting and collaborating on projects.

  • Repositories - the basic unit of GitHub, most commonly a single project
    - location where all the files for a particular project are stored

  • Branching - the way to work on different parts of a repository at one time
    - when creating a branch, you're making a snapshot copy of the original branch
    - used for keeping bug fixes and feature work separate from the master (production) branch
    - when a feature or fix is ready, it is merged into master

  • Commits- saved changes
    - commit message - a description explaining why a particular change was made

  • Issues - a note on a repository about something that needs attention (e.g. bug, feature request, question)

  • Pull Requests - proposing changes and requesting someone pull in your contribution (merge into branch)
    - pull requests can be issued to share screens or ideas, when stuck and need advice, or when ready for review
    - diffs - the changes, additions and subtractions, shown in green and red
    - in the pull request title, add "fixes #3" or "closes #4" with the issue # to close that issue when merging
    - use @mention in Pull Request message to ask for feedback from specific people or teams
