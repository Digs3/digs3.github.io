SASS (CSS with superpowers)

Official Website

SASS extends upon CSS by including support for variables, nested rules, mixins, inline imports, control directives & expressions.

SASS uses SCSS syntax; an extension of the syntax of CSS

Variables

        $font-stack:    Helvetica, sans-serif;
        $primary-color: #333;

Nesting

CSS rules can be nested within one another.

        nav {
          ul {
            margin: 0;
            padding: 0;
            list-style: none; }
          li { display: inline-block; }
          a {
            display: block;
            padding: 6px 12px;
            text-decoration: none; }
        }

Mixins

Mixins are reusable chunks of CSS.

Create a mixin with the @mixin directive

        @mixin boxy($bWidth, $bStyle, $bColor) {
          border-width: $bWidth;
          border-style: $bStyle;
          border-color: $bColor; }

Use the mixin with the @include directive

        $box-style1: 5px, solid, red;
        .first {
          @include boxy($box-style1...); }

Partials & Inline Imports

Partials are Sass files that contain snippets of CSS that you can include in other Sass files. Use snippets to modularize your CSS and keep things easier to maintain.

Partials are named with a leading underscore (e.g., _partial.scss)

Import a partial into another Sass file with the @import directive.

@import 'pageContact'; will import the _pageContact.scss file and combine it with the current file

Extend/Inheritance

Share a set of CSS properties from one selector to another using the @extend directive.

        .message {
          border: 1px solid #ccc;
          padding: 10px;
          color: #333; }
        
        .success {
          @extend .message;
          border-color: green; }

Operators

Sass includes standard math operators like +, -, *, /, and %.

        article {
          float: left;
          width: 600px / 960px * 100%; }
