JavaScript

The programming language of HTML and the Web.

Learn JavaScript

What is JavaScript?

JavaScript is an object-oriented programming language. It is most often used to add client-side behavior to Web pages but in recent years has expanded to the server side thanks to Node.js.

JavaScript remained largely the same for many years until 2015 when ECMAScrcript 2015 (ES2015) was released. ES2015 introduced new ways to declare variables, a syntax for defining classes, modules, promises, and more.

Numbers

Math is a built-in object that performs mathematical tasks on numbers

parseInt(string, radix) is a built-in function that parses a string and returns an integer. Radix is the numeral system to use.

parseFloat(string) parses a string and returns a floating point number.

NaN is a global property whose value is Not-A-Number.

isNaN(testValue) determines whether a value is NaN or not.

numObj.toFixed(digits) rounds a number to the specified decimal point (Default 0)

Strings

Find the length of a string with the length property.

str.charAt(index) returns the specified character from a string.

str.replace(regexp|substr, newSubStr|function) returns a new string with some or all matches of a pattern replaced by a replacement.

        				var newstr = str.replace('apples', 'oranges');

str.toUpperCase();

Operators

Equality Comparisons

The == operator performs type coercion and will result in 1 == true and 123 == "123" being true

The === operator will avoid type coercion
There are also the != and !== operators.

Ternary Operator

var allowed = (age > 18) ? "yes" : "no";

Variable Declaration

Variables can be defined with the var, const, or let keywords.

const and let are scoped to the nearest block (e.g., for, and if statements.)

A variable declared with const cannot be reassigned. A const represents a constant reference to a value, not as a 'constant value'. This means you could set a const to an array and then add items to the array.

A variable declared with let can be reassigned.

The var keyword should be avoided in new code as the other two options provide more detail. These variables are scoped within its nearest parent function.

Objects

Objects can be thought of as simple collections of name-value pairs.

All objects have a prototype property that is itself an object. Objects inherit their properties and methods from their prototype. Attach properties and methods to an objects prototype to make them available to instances of that object (function).
Think of using the prototype property to create a class.

Creating Objects

There are several ways to create objects in JavaScript. You can create an object using a constructor function, using object literals, or using the ES2015 class syntax.

Creating an object using a constructor function

        				function Person(name, age) {
        					this.name = name;
        					this.age = age;
        				}
        				Person.prototype.getInfo = function() {
        					return this.name + ' is ' + this.age + ' years old.';
        				}
        				var you = new Person("You", 24);
        				alert(you.getInfo());

Create an object using an object literal

        				var Person = {
        					name = name;
        					this.age = age;
        				};

Create and initialize an object with object literal syntax
Simple, but every property needs to be specified as there is no class.
Good for one time objects, singleton objects, or objects that act as hashes.

        				var guy = {
        				  name: "John",
        				  "awesome": true,
        				  aboutMe: function() {
        				    return this.awesome ? "I'm awesome!" : "I'm lame :("; }
        				};

Create an object using ES2015 classes.

While JavaScript does conceptually support classes (objects inherit from other objects), ES2015 provides the class syntax to allow for cleaner code. Behind the scenes it is just setting up the prototpye inheritance between objects. Any functions defined in the class definition will exist on the prototype object. 

        					class Person {
        						constructor(name, age) {
        							this.name = name;
        							this.age = age;
        						}
        						getInfo() {
        							return this.name + ' is ' + this.age + ' years old.';
        						}
        					}
        					var you = new Person();

Object Inheritance

Objects can inherit from other objects in JavaScript by configuring prototype inheritance or by using the ES2015 extends keyword. The following examples produce the exact same object structure.

Configuring prototype inheritance

The old way of creating object inheritance is done by configuring the prototype of the inherited object. 

        					function Person(name, age) {
        						this.name = name;
        						this.age = age;
        					}
        					Person.prototype.getInfo = function() {
        						return this.name + ' is ' + this.age + ' years old.';
        					}
        					function Student(studentId, name, age) {
        						this._super.call(this, name, age);
        						this.studentId = studentId
        					}
        					Student.prototype = Object.create(Person.prototype);
        					Student.prototype.constructor = Student;
        					Student.prototype._super = Person;
        					Student.prototype.getStudentInfo = function() {
        						return 'Student ' + this.studentId + ' is ' + this.name + ' and is ' + this.age + ' years old.';
        					}
        					var student = new Student(1, "Bob", "Smith");

Inheritance with ES2015 extends and super keywords

        					class Person {
        						contructor(name, age) {
        							this.name = name;
        							this.age = age;
        						}
        						getInfo = function() {
        							return this.name + ' is ' + this.age + ' years old.';
        						}
        					}
        					class Student extends Person {
        						constructor(studentId, name, age) {
        							super(name, age);
        							this.studentId = studentId
        						}
        						getStudentInfo = function() {
        							return 'Student ' + this.studentId + ' is ' + this.name + ' and is ' + this.age + ' years old.';
        						}
        					}
        					var student = new Student(1, "Bob", "Smith");

Using Objects

Once created, object properties can be accessed in one of two ways

        				obj.name = "Simon";
        				obj["name"] = "Simon";

Check if property exists on an object with the in operator

        				if ("schoolName" in school) {
        				  console.log("School Name: " + school.schoolName); }

Use the hasOwnProperty() method to determine if an object has a specific property as one of its own.

console.log(school.hasOwnProperty("schoolName"));  // true​

Using Getter/Setter Properties

Getter and setter properties are not new to JavaScript.

        					class Person {
        						constructor(firstName, lastName) {
        							this._firstName = firstName;
        							this._lastName = lastName;
        						}
        						get firstName() {
        							return this._firstName
        						}
        						set firstName(value) {
        							this._firstname = value;
        						}
        					}

ES2015 static keyword

Static functions are primarly used as helper functions related to the class. These functions are defined on the class itself, not the prototype; therefore it is not inherited. 

        					class Person {
        						constructor(firstName, lastName) {
        							this._firstName = firstName;
        							this._lastName = lastName;
        						}
        						static getFormattedName(person) {
        							return person.lastName + ", " + person.firstName;
        						}
        					}

Object Design Patterns

Singleton Pattern

Use this pattern when you only want one object to be created of a specific type

Simple Example

        				function Hero(name) {
        					// If Object has already been created
        					if(typeof Hero.instance === 'object') {
        						return Hero.instance; }
        					this.name = name;
        					Hero.instance = this;
        					return this; }

Factory Pattern

Used to generate different objects on request.

        				function Sword(desc) {
        					this.weaponType = "Sword";
        					this.metal = desc.metal || "Steel";
        					this.style = desc.style || "Longsword";
        					this.hasMagic = desc.hasMagic || false;
        				}
        				function Bow(desc) {
        					this.weaponType = "Bow";
        					this.metal = desc.metal || "Wood";
        					this.style = desc.style || "Longbow";
        					this.hasMagic = desc.hasMagic || false;
        				}
        				function WeaponFactory() {};
        				WeaponFactory.prototype.makeWeapon = function(desc) {
        					var weaponClass = null;
        
        					if (desc.WeaponType === "Sword") {
        						weaponClass = Sword;
        					} else if (desc.WeaponType === "Bow") {
        						weaponClass = Bow;
        					} else
        						return false;
        					}
        					return new weaponClass(desc);
        				}
        
        				var myWeaponFact = new WeaponFactory();
        				var bladeFist = myWeaponFact.makeWeapon({
        					weaponType: "Sword",
        					metal: "Dark Iron",
        					style: "Scythe",
        					hasMagic: true
        				})
        				document.write(bladeFist.weaponType + " of type " +
        				  bladeFist.style + " crafted from " + bladeFist.metal + "
");

Arrays

Arrays are objects used to store multiple values.

length returns one more than the highest index (not necessarily the # of items)

        				// Create an array
        				var a = new Array();
        				a[0] = "dog";
        				a[1] = "cat";
        				a.length; // Returns 2
        
        				// Create an array with an array literal
        				var b = ["dog", "cat", "hen"];
        
        				// Loop through array
        				for (var i = 0; i < a.length; i++) {
        				  // Do something with a[i]
        				}
        
        				// Loop through array with forEach()
        				b.forEach(function(currentValue, index, array) {
        				  // Do something with currentValue or array[index]
        				});
        
        				// Add item to array
        				b.push("hamster");

Functions

Functions have access to the arguments variable--an array-like object holding all values passed to the function.

Most basic function

        			function add(x, y) {
        			  var total = x + y;
        			  return total; }

Anonymous Function

        			var avg = function() {
        			  var sum = 0;
        			  for (var i = 0, j = arguments.length; i < j; i++) {
        			    sum += arguments[i];
        			  }
        			  return sum / arguments.length; };

The arrow function syntax is a shorthand for the function syntax

        					let AddTwoNumbers = (x, y) => {return x + y}

Modules

Modules were introduced in ECMAScript 2015. Before this, modules were implemented via libraries, not built into the langugage. Each module is defined in its own file. Functions or variables defined in a module are not visible outside unless you explicitly export them.

Creating Modules

Create a new module by defining a new .js file that contains the variables and functions for the module. Use the export keyword to export the module's functions. Use the as keyword to rename the values while exporting. 

        					function generateRandom() {
        						return Math.random();
        					}
        					function sum(a, b) {
        						return a + b;
        					}
        					export {generateRandom as random, sum}

You can also use the export keyword like this: 

        					export function generateRandom() {
        						return Math.random();
        					}
        					export function sum(a, b) {
        						return a + b;
        					}

Named exports vs default exports

There are two kinds of exports: named exports (several per module) and default exports (one per module). It is possible to use both at the same time, but usually best to keep them separate. The previous examples used named exports.

Every module can have one, and only one, default export that can be exported and imported without specifying the variable name. 

        					// hello-world.js
        					export default function() {}
        				
        					// main.js
        					import helloWorld from './hello-world';

Importing Modules

Modules can be imported into other .js files using the import keyword. You can import only the values you want or the entire module.

Importing specific values

        					import {random, sum} from 'utility';

Importing the entire module

The entire module is imported as an object with properties that relate to the module's values. 

        					import 'utility' as utils;
        
        					console.log(utils.sum(1,2));
        				
        					import * as utility from './utility';
        
        					console.log(utility.sum(1,2));
